# Internationalization (i18n) – Spec

Goals
- Start using language tags immediately; no hardcoded UI strings in new code.
- Ship a complete English fallback in-app; enable dynamic loading of other locales at runtime.
- Allow multiple providers (bundle → HTTP/CDN → IPFS gateway), with caching and integrity checks.
- Keep tests stable by using testIDs and avoiding locale‑specific text assertions.

Decisions
- Library: i18next with react‑i18next; device detection via react‑native‑localize.
- Live switching: implemented now via a lightweight I18nProvider/useT hook (no refactor later).
- Key style: dot‑scoped, kebab for leaf keys, e.g. screens.connections.empty-message.
- Namespaces: common (shared), screens (per‑screen overrides may live under screens.<ScreenName>.*).
- Locales: bundled en and es are available now; others may load OTA later.
- Fallbacks: defaultLocale=en is bundled; es is bundled; other locales may load OTA.
- Deep link override: support query param ?locale=<tag> for demos/tests.

Developer usage
- Use `useT()` hook in components and call: t('scope.key', 'Default English text').
- Do not concatenate translatable strings in code; use interpolation, e.g. t('common.count', '{{n}} items').
- Keep testIDs stable across locales; Detox should assert by id, not by text.

Runtime loading (priority order)
1) Bundled JSON (guaranteed): src/i18n/locales/en/*.json
1b) Bundled JSON (Spanish): src/i18n/locales/es/*.json
2) HTTP(S) backend: GET {baseUrl}/{locale}/{namespace}.json
3) IPFS (optional): ipfs://CID mapped via gateway (https://ipfs.io/ipfs/<CID>)

Caching and integrity
- Cache fetched dictionaries (per locale/namespace) in AsyncStorage with version/ETag.
- Validate JSON with a minimal schema (string values only); reject malformed packs.
- Optional: store a sha256 for each pack; verify on load when provided.

Locale selection
- Initial: device locale (react‑native‑localize) if supported, else defaultLocale.
- Overrides (in order): deep link ?locale=xx, persisted user choice, device locale.
- Persist explicit user choice in AsyncStorage; deep link affects only current session unless persisted.

RTL and formatting
- Mirror layout using I18nManager for rtl locales (ar, he, …).
- Use Intl for numbers/dates; keep formats defined in messages or centralized format helpers.

File layout (convention)
- src/i18n/
  - index.ts (i18n init, backends, language detector)
  - locales/
    - en/
      - common.json
      - screens.json

Failure modes
- If a remote pack fails to load/validate, log and fall back to bundled en.
- Missing keys: show default string from t('key', 'Default English text'); log once per key in dev.

Testing
- Unit: locale parsing and loader selection. Mock backends; ensure fallback works.
- Detox: deep link with ?locale=es and assert a known label via testID.
- Component: render with i18next provider at specific locale; snapshot on testIDs, not text.

Open items
- Define remote endpoints (base URLs / gateway) in env or spec once decided.
- Add contribution guide for translators (JSON/PO export if needed) when multiple locales are added.
