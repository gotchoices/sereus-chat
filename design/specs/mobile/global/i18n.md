# i18n

## Decisions

- **Library:** i18next + react-i18next; device detection via react-native-localize
- **Locales:** en (default, bundled), es (bundled); others may load OTA
- **Namespaces:** common, screens
- **Key style:** dot-scoped, kebab leaf keys (e.g. `screens.connections.empty-message`)
- **RTL:** supported (I18nManager)
- **Deep link override:** `?locale=<tag>` for demos/tests

## Runtime sources (priority)

1. Bundled JSON (guaranteed fallback)
2. HTTP/CDN
3. IPFS gateway (optional)
